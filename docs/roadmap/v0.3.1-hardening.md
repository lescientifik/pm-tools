## Phase 2 — Security Hardening

**Objectif**: Close path traversal, URL injection, thread safety, and DOI encoding vulnerabilities.

**Dépendances**: None (independent of Phase 1)

### 2.1 — Input validation: two-tier strategy (MAJOR 3)

`download.py:304`, `cache.py:87` — identifiers used directly in file paths.

The validation must be **two-tier** because different commands accept different ID formats:
- **E-utilities commands** (`fetch`, `cite`, `search`): PMIDs are strictly numeric.
- **Filesystem commands** (`download`, `cache`): accept PMIDs and PMC IDs (e.g., `PMC1234567`). Need path-safe validation (reject `/`, `..`, null bytes).

**TDD Steps**:

- **RED**: Add tests for `validate_pmid(id)`:
  - `"12345678"` → OK
  - `"../../etc/passwd"` → raises `ValueError`
  - `"PMC1234567"` → raises `ValueError` (numeric-only for E-utilities)
- **RED**: Add tests for `validate_filename_safe(id)`:
  - `"12345678"` → OK
  - `"PMC1234567"` → OK
  - `"../../etc/passwd"` → raises `ValueError`
  - `"foo\x00bar"` → raises `ValueError`
- **RED**: Add entry-point tests for **each** CLI that accepts PMIDs:
  - `fetch.main()` with `"../../etc/passwd"` → reject
  - `cite.main()` with `"../../etc/passwd"` → reject
  - `download.main()` with `"../../etc/passwd"` → reject
  - Note: `refs.main()` is excluded — it accepts file paths, not PMIDs.
- **GREEN**: Create validation functions in `io.py`. Call `validate_pmid()` at E-utilities entry points (`fetch.main`, `cite.main`). Call `validate_filename_safe()` in `cache_write()` (defense in depth) and `download` file path construction.
- **REFACTOR**: None expected.

### 2.2 — URL parameter encoding (MAJOR 4)

`fetch.py:74-75`, `cite.py:34-35` — f-string URL construction.

**TDD Steps**:

- **RED**: Add test for `_fetch_batch(["123", "456"])`: capture the URL passed to `client.get()`, parse it with `urllib.parse.parse_qs()`, assert `id` param is `"123,456"` (literal commas preserved) and `retmode` is `"xml"`.
- **RED**: Same test for `_cite_batch()`, assert `id` param is `"123,456"`.
- **RED**: Add test with malicious PMID `"123&retmode=json"`: parse URL, assert only one `retmode` key exists and its value is `"xml"`.
- **GREEN**: Sanitize individual PMID values (strip whitespace, reject non-numeric via `validate_pmid()` from 2.1) before joining with commas. Keep f-string URL construction — the sanitized PMIDs are safe to interpolate. **Do NOT use `httpx` `params=` dict** — it percent-encodes commas, which breaks E-utilities batch `id` parameter.
- **REFACTOR**: None expected.

### 2.3 — Thread-safe HTTP singleton (MAJOR 5)

`http.py:12-29` — race condition on `_client` global.

**TDD Steps**:

- **RED**: Add test in `test_http.py` using `unittest.mock.patch` to inject a `time.sleep(0.1)` delay in `httpx.Client.__init__`, then spawn 20 threads with a `threading.Barrier` to force simultaneous entry. Assert all threads get the same instance. This test will fail without the lock (the delay widens the race window).
- **GREEN**: Add `threading.Lock` with double-checked locking:
  ```python
  _lock = threading.Lock()
  def get_client(timeout=_DEFAULT_TIMEOUT):
      global _client
      if timeout != _DEFAULT_TIMEOUT:
          return httpx.Client(timeout=timeout, follow_redirects=True)
      if _client is None:
          with _lock:
              if _client is None:
                  _client = httpx.Client(timeout=timeout, follow_redirects=True)
      return _client
  ```
- **REFACTOR**: None expected.

### 2.4 — DOI URL encoding (MAJOR 10)

`download.py:167` — only escapes `/` in DOI for Unpaywall URL.

**TDD Steps**:

- **RED**: Add test: DOI `"10.1234/foo#bar"` → URL should contain `%23` not `#`.
- **RED**: Add test: DOI `"10.1234/foo?bar&baz"` → properly encoded.
- **GREEN**: Replace `doi.replace("/", "%2F")` with `urllib.parse.quote(doi, safe="")`.
- **REFACTOR**: None expected.

**Parallélisation**: 2.1 must land first (creates `validate_pmid()` used by 2.2). Then 2.2, 2.3, 2.4 can run in parallel → **2.1 sequential, then 3 parallel OPUS agents**.

**Critères de complétion**:
- [ ] Two-tier validation: numeric for E-utilities, path-safe for filesystem
- [ ] Validation at all entry points + defense in depth in `cache_write()`
- [ ] URL params properly encoded in fetch + cite (commas preserved)
- [ ] HTTP singleton is thread-safe (deterministic test proves it)
- [ ] DOI special characters properly encoded

## Phase 5 — Boilerplate & Style

**Objectif**: Reduce repetitive patterns and fix remaining style issues.

**Dépendances**: Phase 4 (init.py argparse migration must complete first)

### 5.1 — Extract `safe_parse()` helper (STYLE 19)

9+ files have a SystemExit catch after `parser.parse_args()`. Most return `int(e.code)`, but `collect_main` and `diff.main` hardcode `return 2`.

**Design**: `safe_parse(parser, args) -> argparse.Namespace` that raises a custom `ParseExit(code: int)` exception on SystemExit. Each caller catches `ParseExit` and returns the code. This preserves the existing behavior: callers that want to remap (e.g., `collect_main` returning 2) can still do so.

Alternative: `safe_parse(parser, args) -> tuple[Namespace | None, int | None]` — returns `(namespace, None)` on success, `(None, exit_code)` on failure. Callers check the second element.

**TDD Steps**:

- **RED**: Add test for `safe_parse(parser, args)`:
  - Valid args → returns `(Namespace, None)`
  - `--help` → returns `(None, 0)`
  - Invalid args → returns `(None, 2)`
- **GREEN**: Create helper in `io.py`. Replace all call sites, preserving each caller's specific exit-code behavior.
- **REFACTOR**: Verify all 9+ subcommand `main()` functions still behave identically by running full test suite.
